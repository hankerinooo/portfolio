# Portfolio — Claude Code Context

## Project Overview

Static portfolio site for Henry Davis. The `index.html` is a hand-written placeholder page. The real output is generated by `build.js`, a zero-dependency Node 18+ script that reads from a Notion database and writes `_site/index.html` and `_site/projects/{slug}.html`. Deployment is handled by a GitHub Actions workflow that runs on push to `main` and on an hourly cron.

**Important:** CSS is defined inline in two places — inside `index.html` and inside `build.js`. Any style changes must be made in both files.

## Design Tokens

All tokens are CSS custom properties defined in `:root`. Do not hardcode hex values — always reference these variables.

### Colors

| Variable | Role |
|---|---|
| `var(--bg)` | Page background |
| `var(--ink)` | Primary text / foreground |
| `var(--ink-mid)` | Mid-opacity ink (borders, muted UI) |
| `var(--ink-faint)` | Low-opacity ink (dividers, subtle backgrounds) |

Dark mode is activated by `html[data-theme="dark"]` (explicit toggle) or `@media (prefers-color-scheme: dark)` (system preference). The palette is warm monochromatic — cream/off-white in light mode, deep warm dark in dark mode. There is no accent color.

### Typography

| Usage | Value |
|---|---|
| Headings | `'Fraunces', Georgia, serif` — weight 400, optical-sizing auto |
| Body / UI | `'Jost', system-ui, sans-serif` — weights 300, 400, 500 |

Body font size is `1.0625rem`, line-height `1.65`. Headings use tight letter-spacing (`-0.025em` to `-0.03em`).

## Doom Mode

An ongoing feature. The full implementation lives in a single new file: **`doom.js`**.

### Changes to existing files

The only modifications allowed to existing files are:

1. **`index.html`** — add `<script src="doom.js"></script>` immediately before `</body>`
2. **`index.html`** — add one trigger link inside `<footer>`, after the existing copyright line

### Trigger link spec

```html
<a href="#" class="doom-trigger" onclick="window.initDoomMode();return false;">Answer the call to defeat Dark Patterns?</a>
```

Style requirements:
- `opacity: 0.15` — nearly invisible by default
- Uses existing body font styles (inherits `font-family`, `font-size`, `color: var(--ink)`)
- `text-decoration: none` by default
- `text-decoration: underline` on `:hover`
- No navigation — calls `window.initDoomMode()` on click

### doom.js spec

- Exposes a single global: `window.initDoomMode()`
- Renders a first-person ASCII raycaster on a full-screen `<canvas>` overlay that sits above all page content
- Visual style: uses the site's `--bg` and `--ink` CSS variables exclusively — no images, no sprites, ASCII characters only
- Enemies: UX anti-patterns personified (dark patterns, manipulative UI, etc.)
- Tone: the portfolio itself mutates into a game — the aesthetic should feel continuous with the site, not like an injected foreign element
- Escape or a defined key closes the overlay and returns to the portfolio
